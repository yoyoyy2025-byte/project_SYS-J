# 📂 Job-Navigator 데이터 수집 리스트

이 프로젝트의 RAG(검색 증강 생성) 모델 학습을 위한 고품질 취업 데이터 수집 목표입니다.
수집된 데이터는 `career_data.py` 형식에 맞춰 정제됩니다.

---

## 1. 📝 자소서/이력서 작성 가이드 (Methodology)
> **목표:** AI가 자소서의 논리적 구조를 판단하고 조언할 수 있게 함.

- [ ] **STAR 기법 가이드**
    - 상황(S), 과제(T), 행동(A), 결과(R) 작성 요령 및 예시
- [ ] **두괄식 작성법**
    - 핵심 성과를 문단 처음에 배치하는 방법과 나쁜 예시 비교
- [ ] **수치화 원칙**
    - 정성적 표현("많이", "열심히")을 정량적 수치("20% 증가", "3개월 단축")로 바꾸는 법
- [ ] **소제목 활용법**
    - 인사담당자의 눈길을 끄는 소제목(Headline) 작성 패턴

---

## 2. 💼 직무별 핵심 역량 키워드 (Keywords)
> **목표:** 각 직무 지원자가 반드시 포함해야 할 전문 용어와 기술 스택 학습.

### 💻 개발자 (Backend/Frontend/AI)
- [ ] **기술 스택:** Python, Java, Spring, React, Next.js, AWS, Docker 등
- [ ] **협업 툴:** Jira, Slack, Notion, Git/GitHub 사용 경험
- [ ] **핵심 역량:** 트러블 슈팅, 대용량 트래픽 처리, 리팩토링, 코드 리뷰, 성능 최적화

### 🎨 디자이너 (UI/UX/Web)
- [ ] **툴:** Figma, Sketch, Photoshop, Illustrator, Protopie
- [ ] **핵심 역량:** 사용자 리서치, 와이어프레임, 프로토타이핑, 디자인 시스템, A/B 테스트

### 📈 마케터/기획자 (Performance/Content/PM)
- [ ] **툴:** GA4(Google Analytics), SQL, Amplitude, Excel/Spreadsheet
- [ ] **핵심 역량:** 데이터 분석, ROAS 개선, KPI 설정, 커뮤니케이션, 일정 관리

---

## 3. ❌ vs ⭕ 첨삭 예시 (Before & After)
> **목표:** AI가 구체적인 수정 제안을 할 수 있도록 학습시킴. (가장 중요 ⭐)

- [ ] **지원동기**
    - Bad: "회사가 좋아서 지원했습니다."
    - Good: "귀사의 OO 서비스 런칭에 제 OO 경험을 기여하고 싶어 지원했습니다."
- [ ] **성격의 장단점**
    - Bad: "너무 꼼꼼해서 시간이 오래 걸립니다."
    - Good: "꼼꼼함 덕분에 오류를 줄이지만, 속도 개선을 위해 체크리스트를 활용하고 있습니다."
- [ ] **실패/극복 사례**
    - Bad: "프로젝트가 실패해서 아쉬웠습니다."
    - Good: "초기 기획 실패로 프로젝트가 엎어졌으나, 원인을 분석해 프로세스를 재정립하고 결국 완수했습니다."

---

## 4. 🎤 면접 예상 질문 & 답변 전략 (Interview)
> **목표:** 자소서 이후 면접 코칭까지 확장 대비.

- [ ] **공통 질문:** 1분 자기소개, 마지막으로 하고 싶은 말, 공백기 질문
- [ ] **직무 질문:** (개발) REST API란?, (디자인) 포트폴리오 기여도는?
- [ ] **컬처핏 질문:** 갈등 해결 경험, 야근에 대한 생각, 리더십 경험

---

## 5. 🏢 기업별 인재상 및 트렌드 (Trend)
> **목표:** 특정 기업 지원 시 맞춤형 조언 제공.

- [ ] **네카라쿠배 (IT 대기업)**
    - 코드 퀄리티, 자기 주도적 성장, 공유 문화 강조
- [ ] **삼성/LG (대기업)**
    - 조직 융화, 성실함, 직무 기초 역량 강조
- [ ] **스타트업**
    - 문제 해결 능력, 빠른 실행력, 올라운더 기질 강조
